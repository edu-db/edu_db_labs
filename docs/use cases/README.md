# Модель прецедентів

## 1. Діаграма прецедентів

@startuml
'default
top to bottom direction

Клієнт -left-> (Авторизуватися)
Клієнт -right-> (Зареєструватися)
Клієнт <-down- Експерт
Експерт--> (Голосувати)
Клієнт <-- Користувач
Користувач --> (Створити опитування)
Користувач --> (Редагувати опитування)
Користувач --> (Видалити опитування)
Користувач --> (Отримати результати опитування)

@enduml

## 2. Схема використання для Клієнта

@startuml
'default
top to bottom direction

Клієнт -left-> (Авторизуватися)
Клієнт -right-> (Зареєструватися)
(Зареєструватися за допомогою:\n gmail, telegram, linkedin...) .left.> (Зареєструватися) : extends
(Відновити пароль) .> (Авторизуватися) : extends
@enduml

## 3. Схема використання для Експерта

@startuml
'default
top to bottom direction

Експерт--> (Голосувати)

@enduml

## 4. Схема використання для Користувача

@startuml
'default
top to bottom direction

Користувач --> (Створити опитування)
Користувач --> (Редагувати опитування)
Користувач --> (Видалити опитування)
Користувач --> (Отримати результати опитування)
(Створити посилання) .up.> (Створити опитування) : extends
(Налаштувати доступ) .up.> (Створити опитування) : extends
(Перенести до кошика) .down.> (Видалити опитування) : extends
(Видалити відразу) .down.> (Видалити опитування) : extends
(Форматувати результати \nу різні розширеня) .up.> (Отримати результати опитування) : extends
(Аналіз результатів) .up.> (Отримати результати опитування) : extends
@enduml
