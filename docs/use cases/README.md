# Модель прецедентів

## Загальна діаграма

Основні можливості з груп користувачів (Діаграма №1)


@startuml

    actor "Користувач" as User #black
    actor "Адміністратор" as Admin #black

    User  -d-|> Admin

    usecase "Реєстрація" as Registration

    usecase "Авторизація" as Login 
    usecase "Управління \nданими" as Search
    usecase "Редагування \n профілю" as Edit
    usecase "Використання \nіснуючих матеріалів" as Sources
    usecase "Звернення \nза допомогою" as Help

    usecase "Додавання \nінформації" as AddSource
    usecase "Редагування \nбази користувачів" as Users
    usecase "Отримання \nстатистики" as Statistic

    User -u-> Registration
    User -u-> Login
    Registration .l.> Login
   
    note left of Login #ffc
        <i>Якщо була 
        <i>проведена 
        <i>реєстрація
    end note

    User -l-> Search
    User -d-> Edit
    User --> Help
    User -r-> Sources

    Admin -d-> AddSource
    Admin -d-> Users
    Admin -l-> Statistic

@enduml


Можливості користувача (Діаграма №2)


@startuml

    actor "Користувач" as User #black

    usecase "Звернення \n за допомогою" as Help
    usecase "До системи" as SystemHelp
    usecase "До адміністратора" as AdminHelp

    usecase "Управління даними" as Sources
    usecase "Додати текст" as Add
    usecase "Видалити \nдоданий текст" as Remove
    usecase "Запит на \nобробку даних" as Accept

    AdminHelp .u.> Help
    SystemHelp .d.> Help

    Add .l.> Sources
    Remove .u.> Sources
    Accept .d.> Sources

    User -l-> Help
    User -r-> Sources

@enduml


### USER.REGISTRATION

```
НАЗВА: Авторизувати користувача і створити йому обліковий запис;
УЧАСНИКИ: Користувач, Система;                          
ПЕРЕДУМОВИ: Користувач повинен  мати бажання зареєструватися і користуватися системою;
РЕЗУЛЬТАТ: Користувач отримує власний обліковий запис, отримує доступ до облікового запису;
ВИКЛЮЧНІ СИТУАЦІЇ: Користувач не має бажання реєструватися;
ОСНОВНИЙ СЦЕНАРІЙ:                                      
```


@startuml

|#lightgrey|Користувач| 
    start
    :Натискає на кнопку входу \nдля реєстрації;
|#darkgrey|Система|
    :Отримує запит на реєстрацію;
    :Надсилає форму для вводу \nданих створення облікового \nзапису;
|Користувач|
    :Вводить дані для створення \nоблікового запису;
    :Натискає кнопку надсилання \nсистемі даних для реєстрації;
|Система|
    :Отримує дані користувача;
    :Підтверджує дані;
    :Створює запис про \nдодавання користувача;
    :Додає користувача \nдо бази;
|Користувач| 
    :Отримує доступ до \nоблікового запису;
    stop

@enduml

### USER.GET_DATA

ВИКЛЮЧНІ СИТУАЦІЇ: Користувач не є зареєстрованим на сайті, у форми для заповнення нічого заданого;

    stop

@enduml
